Проверь все, что мы сейчас положили в архив — найди export_path из вывода команды archive-session в этом разговоре. Если в $ARGUMENTS передан путь или session_id — используй его. Если ни того ни другого нет — спроси, не угадывай.

Прочитай экспортный `.md` файл целиком.

**ВАЖНО: ты видишь весь этот разговор в своём контексте. Это твой источник правды — не парсер, не скрипт. При проверке каждого элемента сравнивай содержимое архива с тем, что реально происходило в этом чате по твоим наблюдениям.**

Затем проверь **КАЖДЫЙ** элемент по этому чеклисту:

**Summary** — точно описывает работу этой сессии? Не предыдущей, не из HANDOFF?

**Model** — конкретный ID (`claude-sonnet-4-6`), не просто `claude`?

**Tasks** — только задачи, с которыми реально работали? Не из прочитанных HANDOFF/контекста?

**Tags (domain/skill)** — только реально использованные? Не из system-reminder, не из XML-маркеров `<command-message>`, `<command-name>`?

**Events** — каждое событие подтверждено реальным tool call? (`changelog_read` — был ли Read CHANGELOG-файла, не просто упоминание?)

**Artifacts** — все файлы реально читались/менялись? Не из grep-вывода, не субагентские?

**Diffs — проверить по трём осям (обязательно, не пропускать):**
1. **Набор файлов** — совпадает с тем, что реально трогали в этой сессии?
2. **Контент** — добавления/удаления соответствуют реальным правкам? Нет лишних/пропущенных изменений?
3. **Формат** — нет ли: двойных слешей `a//`, пустых строк между каждой строкой кода, ложных `-`/`+` у неизменённых строк, неверных номеров `@@ -1,`?

После проверки — пофикси все найденные ошибки в записи в архиве (export `.md` + SQLite). Не скрипт, а запись в архиве.

Найди в is/personal папку архивирующего проекта, в ней найди доку с багами и добавь туда найденные баги и рекомендации по фиксам.
