# SA-T074 — Tasks: ложные T-номера из шаблона /smoketest

## Проблема

Парсер задач извлекает T-номера из **шаблона скилла** `/smoketest`, а не из реальной работы сессии.

Шаблон содержит примеры: "T032 X1: NewTaskRun", "T048 (PipelineEngine) вызывает T024 (PhaseExecutor)", "T042", "T099", "T001" — парсер считает их реальными задачами сессии.

## Воспроизведение

Сессия `33b531d7` работала только с T152. В архив попали: T001, T024, T032, T042, T048, T099, T152.

## Root cause

`extract_tasks()` не фильтрует T-номера из `role: user` сообщений, которые являются шаблонами скиллов (skill template expansion). Шаблон скилла подставляется как user message с `<command-message>` тегом.

## Рекомендация

Вариант A: Игнорировать T-номера из user messages, содержащих `<command-message>` / `<command-name>` теги — это skill template expansions, не пользовательский ввод.

Вариант B: Считать только T-номера из assistant messages (реальная работа).

Вариант C: Фильтровать T-номера, встречающиеся ТОЛЬКО в шаблонных примерах (code blocks, цитатах) — не в прямом тексте assistant.
