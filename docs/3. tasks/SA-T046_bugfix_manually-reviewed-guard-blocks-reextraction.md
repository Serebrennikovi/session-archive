# SA-T046 — Исправить: флаг manually_reviewed блокирует re-extraction при повторной архивации

## Customer-facing инкремент

При повторном `archive-current` с явным `--summary` для сессии, помеченной как `manually_reviewed=1`, новый summary применяется. Дифы и артефакты также переизвлекаются, если добавить `--force` без уничтожения ручных правок (умный merge).

---

## Баги

### BUG-MANUAL-GUARD-SUMMARY — --summary игнорируется при manually_reviewed=1

**Симптом:** Если сессия была архивирована и вручную отредактирована (`manually_reviewed=1`), повторный вызов `archive-current --summary "новый текст"` выводит:
```
Warning: session <id> is manually reviewed. Updating counts only. Use --force to re-extract all fields.
```
и новый summary НЕ применяется. Поле `summary` в SQLite и в экспортном `.md` остаётся из первой архивации.

**Найден в сессии:** `491d67c9` (2026-04-07) — summary не обновился при повторном `archive-current --summary "..."` несмотря на явный аргумент.

**Root cause:** В `archive-current` логика "manually reviewed guard" блокирует ВСЕ поля включая явно переданный `--summary`. Пользователь не может исправить summary без `--force` (который перезаписывает все ручные правки).

**Ожидаемое поведение:** `--summary "..."` при повторном запуске обновляет только поле `summary`, независимо от `manually_reviewed`. Ручные правки тегов, артефактов и событий не затрагиваются.

**Fix (предлагаемый):**
```python
if session_exists and manually_reviewed:
    if args.summary:
        # Явный summary — применяем даже при manually_reviewed
        update_summary(session_id, args.summary)
    update_counts_only(session_id, msg_count, tool_count)
    return
```

### BUG-DIFFS-CROSS-SESSION — git diff HEAD включает изменения из предыдущих сессий

**Симптом:** `git diff HEAD` в `extract_diffs()` возвращает ВСЕ незакоммиченные изменения репозитория, а не только изменения текущей сессии. Если несколько сессий изменили файлы без коммита между ними, дифы первой сессии включают изменения последующих, и наоборот.

**Найден в сессии:** `491d67c9` — дифы показали T033-T036 (изменены в предыдущих сессиях), не показали T037, T040, T041-T047 (реально изменённые в этой сессии). Сессия была архивирована в середине работы — последующие изменения в дифы не попали.

**Root cause:** Нет механизма для определения какие файлы были изменены именно в конкретной JSONL-сессии. `git diff HEAD` — это diff рабочего дерева vs последний коммит, без привязки ко времени или session_id.

**Возможные fix стратегии:**
1. **По JSONL tool calls:** парсить Edit/Write/Bash tool use в JSONL и извлекать пути файлов — точно, но сложно
2. **По mtime:** сравнивать mtime файлов с `session.created_at` и `session.ended_at` — приближённо, работает без парсинга JSONL
3. **По git stash/snapshot:** делать `git stash` → `git diff HEAD stash` → `git stash pop` — инвазивно
4. **Документировать ограничение:** добавить в README/автоматическое предупреждение в дифах, что `git_head` дифы = весь uncommitted diff, не только сессионный

**Рекомендуемый fix:** Стратегия 1 (JSONL tool calls) для точности; стратегия 4 как немедленный патч (уже частично реализован через примечание в экспортах).

### BUG-FIRST-ARCHIVE-MID-SESSION — Первичная архивация в середине сессии не обновляется

**Симптом:** Если `archive-current` вызван в середине длинной сессии (например, через `/archive-session` после первого этапа работы), а потом сессия продолжилась — итоговый экспорт содержит только данные до первой архивации. При повторном `archive-current` в конце сессии — `manually_reviewed` блокирует обновление дифов и артефактов.

**Найден в сессии:** `491d67c9` — первая архивация сохранила только T033-T036 (ранние изменения), вся основная работа с S07/S08 не попала в дифы.

**Root cause:** Комбинация BUG-MANUAL-GUARD-SUMMARY и BUG-DIFFS-CROSS-SESSION. Когда пользователь хочет обновить архив в конце сессии — нет механизма "дополнить" данные без `--force`.

**Fix (предлагаемый):** Добавить `--append-artifacts` флаг, который добавляет новые артефакты/дифы к существующим без перезаписи ручных правок в тегах/событиях.
