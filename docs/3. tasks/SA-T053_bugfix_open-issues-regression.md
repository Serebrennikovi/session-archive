# SA-T053 — Регрессия: Open Issues снова содержит internal status messages

## Проблема

Повторение SA-T016 (done 2026-04-03). Парсер снова захватывает внутренние статусные сообщения ассистента как "open issues".

## Воспроизведение (2026-04-08, session d62bc616)

Open Issues записано: `"Now let me read the remaining dependencies and run the pre-flight checks."`

Это внутреннее сообщение ассистента между tool calls — не issue и не проблема.

## Корневая причина

Фикс SA-T016 вероятно фильтрует конкретные паттерны (`Удалено .* сессий`), но не ловит generic assistant status messages. Парсер не отличает "ассистент сообщает о действии" от "ассистент указывает на нерешённую проблему".

## Рекомендация по фиксу

Ужесточить фильтр: Open Issues должны содержать ТОЛЬКО строки, которые:
1. Начинаются с маркера проблемы (`ISSUE:`, `BUG:`, `WARN:`, `TODO:`, `FIXME:`)
2. Или находятся в explicit `## Open Issues` секции, заполненной ассистентом
3. Или содержат слова-индикаторы проблем (`не работает`, `сломано`, `нужно исправить`, `blocked`, `failed`)

Строки вида "Let me...", "Now I'll...", "Reading..." — internal monologue, не issues.
