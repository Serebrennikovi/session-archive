# SA-T017 — Исправить фантомные артефакты из соседних сессий

## Customer-facing инкремент

После архивирования сессии в Artifacts будут только файлы, реально затронутые в этой сессии — без "утечки" артефактов из предыдущих сессий, архивировавшихся в том же разговоре.

---

## Баги

### BUG-ARTIFACTS-PHANTOM-CONTEXT — артефакты из предыдущей сессии попадают в текущую

**Симптом:** В `session_artifacts` появляются файлы, которые не читались и не изменялись в текущей сессии:
- `SA-HANDOFF.md (read)` — не читался в сессии
- `ADV-architecture.md (read)`, `config.go (read)`, `ADV-S03 spec (read)` — файлы другого проекта из предыдущей архивации
- `BUGS.md (read)` — не читался, захвачен из контекста предыдущей /verify-archive
- `event:tests × 5` — фантомные записи вида `file_path = "event:tests"` вместо реального артефакта

**Root cause (основной):** Архивный скрипт извлекает tool calls из всего conversational контекста разговора. Если в том же разговоре ранее выполнялась архивация другой сессии (например /archive-session для сессии A, потом /archive-session для сессии B), tool calls сессии A попадают в artifacts сессии B. Нет фильтрации по `session_id` или по временно́му окну текущей сессии.

**Root cause (event:tests):** Artifact-парсер и event-парсер работают независимо. При детекции `go test ...` в Bash команде, артефакт-парсер создаёт запись `file_path = "event:tests"` — что является event, а не артефактом. Записи с `file_path` начинающимся на `event:` нужно фильтровать.

**Повторялось:** 4+ сессии, каждый раз удаляется вручную.

**Fix:**

1. **Фильтрация по времени/session_id:** При извлечении artifacts для сессии `S` использовать только tool calls с `created_at >= session_start_time`. Если session_start_time хранится — применять как нижнюю границу.

2. **Фильтр `event:` артефактов:** При записи `session_artifacts` — если `file_path.startswith('event:')` — пропускать запись (это event, не артефакт).

3. **Дедупликация по session context:** Если архивация запускается несколько раз в одном разговоре (например /accept + /archive-session), каждый раз пересчитывать artifacts только из дельты с момента предыдущей архивации.

---

## Scope

- `session_archive.py`: добавить фильтрацию tool calls по `session_start_time` при сборе artifacts
- Фильтровать `file_path` начинающиеся с `"event:"` при записи в `session_artifacts`
- Тест: архивировать две сессии подряд в одном разговоре — артефакты не должны пересекаться

## Out of scope

- Изменение схемы БД
- Ретроактивная очистка исторических артефактов

---

## Как протестировать

1. Выполнить /archive-session для сессии A, затем для сессии B в одном разговоре
2. Проверить `session_artifacts` сессии B — не должно быть файлов из сессии A
3. Запустить сессию с `go test` → артефакты не должны содержать `event:tests`

## Критерии приёмки

1. Артефакты сессии содержат только файлы с tool calls после `session_start_time`
2. Записей с `file_path LIKE 'event:%'` в `session_artifacts` нет
3. При двух последовательных архивациях в одном разговоре артефакты не пересекаются
