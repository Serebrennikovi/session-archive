# SA-T003 — Исправить BUG-TASKS-FALSE-POSITIVE: фантомные задачи из контекста

## Customer-facing инкремент

После архивирования сессии `session_tasks` содержит только задачи, с которыми реально работали (Edit/Write файла задачи, явное упоминание в tool call), без задач из HANDOFF/system-reminder контекста.

---

## Баг

### BUG-TASKS-FALSE-POSITIVE — T-номера из контекста записываются как выполненные задачи

**Симптом:** В `session_tasks` попадают задачи (T001, T042, T099 и т.д.), которые только упоминались в прочитанных файлах (HANDOFF, архитектура, system-reminder скилл-промпты) — но реально не обрабатывались в сессии. В экспорте Tasks-секция содержит фантомные задачи с action `mentioned`.

**Root cause:** `detect_tasks()` применяет regex `T\d+` ко всему тексту диалога, включая tool_result содержимое (прочитанные HANDOFF, скилл-инструкции, system-reminder). Это захватывает любые T-номера из контекста — задачи других проектов, исторические упоминания, примеры.

**Повторялось:** 24+ раза. Каждый раз вручную удаляется 2–8 фантомных задач.

**Типичный пример:** Работали только с T288, но в Tasks: T284, T285, T287 (из HANDOFF), T042 (из примеров скиллов).

**Fix:**

Засчитывать задачу только при наличии одного из следующих доказательств в tool calls:
1. **Edit/Write** к файлу, содержащему номер задачи в пути (например, `T288_...md`)
2. **Явное упоминание в summary** (передаётся через `--summary` аргумент)
3. **Read к файлу задачи + дальнейший Edit** (прочитали и изменили)

Исключать из обнаружения:
- T-номера из `tool_result` (Read-outputs)
- T-номера из system-reminder и скилл-промптов
- T-номера из сообщений с role `system`

---

## Scope

- Рефактор `detect_tasks()`: ограничить источник T-номеров только tool_use (Edit/Write/Read) и явным summary
- Добавить логику: T-номер засчитывается только если был изменён соответствующий файл задачи
- Action `mentioned` → убрать или сделать opt-in (только при явном `--include-mentioned` флаге)
- Спеки (`S\d+`) исключить из session_tasks полностью

## Out of scope

- BUG-TASKS-EMPTY (Tasks секция пустая когда задача выполнялась) — частично решается этим фиксом
- Ретроактивное исправление уже записанных сессий

---

## Как протестировать

1. Запустить сессию где работали с одной задачей (например T288), при этом HANDOFF содержит ссылки на другие задачи
2. Архивировать: `/archive-session`
3. Проверить: `python3 session_archive.py query "SELECT task_id, actions FROM session_tasks WHERE session_id=(SELECT id FROM sessions ORDER BY created_at DESC LIMIT 1)"`
4. Убедиться что в списке только реально использованные задачи

## Критерии приёмки

1. В `session_tasks` нет задач, которые только читались как контекст в HANDOFF или system-reminder
2. T-номер из прочитанного HANDOFF без Edit/Write к файлу задачи → не попадает в Tasks
3. Спеки (`SA-S01`, `ADV-S03` и т.д.) отсутствуют в `session_tasks`
4. Задача реально выполнявшаяся (Edit к её файлу) → корректно попадает в Tasks
