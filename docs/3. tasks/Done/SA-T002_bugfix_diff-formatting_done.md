# SA-T002 — Исправить форматирование diff-блоков в экспорте

## Customer-facing инкремент

После архивирования сессии diff-блоки в markdown-экспорте читаемы: пути корректные (без двойных слешей), код не растянут пустыми строками, scope diff совпадает с реально изменёнными строками.

---

## Баги

### BUG-DIFF-DOUBLE-SLASH — двойной слеш в заголовках диффов

**Симптом:** `--- a//Users/is/...` и `+++ b//Users/is/...` — лишний слеш в путях.

**Root cause:** При построении заголовка diff к абсолютному пути добавляется `a/` без strip leading `/`:
`f"a/{path}"` → `a//Users/is/...`

**Повторялось:** 20+ раз, в каждой сессии с Edit/Write tool calls.

**Fix:**
```python
f"a/{path.lstrip('/')}"
f"b/{path.lstrip('/')}"
```

---

### BUG-DIFF-EMPTY-LINES — пустые строки между каждой строкой кода в диффе

**Симптом:** В synthetic diff-блоках между каждой `+`/`-` строкой стоит пустая строка — diff нечитаем.

**Root cause:** При сборке synthetic diff к содержимому строк добавляется лишний `\n`, что приводит к `\n\n` вместо `\n`.

**Повторялось:** Каждая сессия с Edit tool calls (50+ раз).

**Fix:**
```python
diff_block = re.sub(r'\n\n([-+ @])', r'\n\1', diff_block)
```

---

### BUG-DIFF-UNTRACKED-SCOPE — untracked файл показан как полностью новый

**Симптом:** Файл, который существовал до сессии (untracked в git), показан как `@@ -0,0 +1,N @@` — весь файл как новый. Реально изменилось несколько строк.

**Root cause:** При отсутствии git-истории скрипт строит diff от пустоты (`"" → current content`), не проверяя Read-snapshots той же сессии.

**Повторялось:** 17+ раз.

**Fix:** При построении synthetic diff для файла без git-истории — искать Read tool call того же пути в текущей сессии и строить diff от этого snapshot. Если Read-snapshot есть → `diff(read_content, write_content)`. Если нет → оставить текущее поведение.

---

## Scope

- Фикс заголовков диффов: `path.lstrip('/')` при сборке `a/` и `b/` префиксов
- Фикс пустых строк: `re.sub(r'\n\n([-+ @])', r'\n\1', diff_block)` перед записью diff
- Фикс untracked scope: использовать Read-snapshot как baseline вместо пустого файла
- Применить ко всем типам diff: `edit_snippet`, `write_snapshot`, `synthetic`

## Out of scope

- BUG-DIFF-WRONG-SESSION (diff из другой сессии) — отдельная проблема
- BUG-DIFF-FAILED-EDIT-CAPTURED (diff от неудачного Edit) — отдельная проблема
- BUG-DIFF-DUPLICATE (дублирующийся diff одного файла) — отдельная проблема
- Исправление уже записанных экспортов

---

## Как протестировать

1. Запустить `/archive-session` в проекте с несколькими Edit-вызовами
2. Открыть markdown-экспорт: `data/exports/YYYY-MM-DD_*.md`
3. Проверить секцию `## Diffs`:
   - Заголовки вида `--- a/Users/is/...` (без двойного слеша)
   - Код идёт строка за строкой без пустых строк между ними
4. Для untracked файла: diff показывает только изменённые строки, не весь файл

## Критерии приёмки

1. `--- a/Users/is/...` — одинарный слеш в заголовках всех диффов
2. В diff-блоке нет пустых строк между соседними `+`/`-` строками
3. Untracked файл с Read-snapshot в сессии → diff от snapshot, не от пустоты
4. Тесты: архивирование сессии с Edit tool calls → diff корректен без ручных правок
