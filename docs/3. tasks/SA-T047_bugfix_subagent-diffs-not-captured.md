# SA-T047 — Субагентские diffs не попадают в экспорт

## Проблема

Парсер JSONL (`session_archive.py`) извлекает diffs только из tool_use вызовов главной сессии. Правки, выполненные через Agent tool (субагенты), записываются в отдельные JSONL-файлы в поддиректории `subagents/` или во вложенные объекты, которые парсер не обходит.

**Пример:** Сессия 269d0bd4 — 29 файлов изменены через 9 параллельных субагентов, но в экспорте только 4 diff-блока из главной сессии.

## Воспроизведение

1. Запустить сессию с `Agent` tool calls
2. Субагенты модифицируют файлы через Edit/Write
3. Архивировать сессию
4. В экспорте: файлы из субагентов отсутствуют в секции Diffs

## Ожидаемое поведение

Diffs из субагентов попадают в экспорт с пометкой `(via subagent)`.

## Рекомендация по фиксу

1. Найти output-файлы субагентов по паттерну `tasks/<agentId>.output` в temp-директории сессии
2. Распарсить каждый `.output` как JSONL — извлечь Edit/Write tool calls
3. Включить в секцию Diffs с пометкой источника

Альтернатива: при Agent tool use в главном JSONL есть task-notification с результатом — можно извлекать список изменённых файлов из summary субагента.
