# Конвенции документации: спецификации и задачи

**Версия:** 1.0
**Дата создания:** 2026-04-01
**Статус:** Действует
**Префикс проекта:** `SA-`

---

## Цель

Установить единые правила именования, структуры и размещения документов проекта для обеспечения:
- Прозрачности и порядка в документации
- Упрощения навигации и поиска
- Согласованности между задачами и спецификациями
- Чёткого отслеживания истории изменений

---

## 0. Префикс проекта

Все нумерованные документы используют префикс **`SA-`** (Session Archive).

Примеры: `SA-BR01_...`, `SA-S01_...`, `SA-T001_...`, `SA-HANDOFF.md`, `SA-CHANGELOG.md`

---

## 1. Именование спецификаций

### Формат
```
SA-SNN_<snake_case_название>.md
```

### Правила
- `NN` — двузначный номер (01, 02, 03, ..., 99)
- Нумерация **хронологическая** (увеличивается с новыми спецификациями)
- Название в `snake_case` (только латиница, строчные буквы, подчёркивания)
- Файл размещается в `docs/2. specifications/`

### Примеры
```
SA-S01_fts_search.md
SA-S02_web_dashboard.md
SA-S03_codex_full_support.md
```

### Статусы спецификаций

- **draft** — черновик, в разработке
- **active** — действующая, по ней ведётся работа
- **done** — завершённая, все задачи выполнены
- **deprecated** — устаревшая, заменена другой спецификацией

Статус указывается в шапке документа:
```markdown
**Статус:** active
```

## 2. Именование задач

### Формат
```
SA-TNN_sNN_<snake_case_название>.md
```

### Правила
- `NN` — двузначный/трёхзначный номер (001, 002, ..., 099, 100, ...)
- Нумерация **сквозная** по всем задачам (в порядке создания)
- `sNN` — номер спецификации, к которой относится задача
- Название в `snake_case` (только латиница, строчные буквы, подчёркивания)
- **Обязательно** в начале каждой задачи добавить строку:
  ```markdown
  **Спецификация:** docs/2. specifications/SA-SNN_<имя>.md
  ```

### Примеры
```
SA-T001_s01_fts_indexing.md
SA-T002_s01_fts_search_api.md
SA-T003_s02_datasette_config.md
```

---

## 3. Определение следующего номера задачи

### Через командную строку:
```bash
find "docs/3. tasks" -name "SA-T*.md" | grep -oE 'T[0-9]+' | grep -oE '[0-9]+' | sort -n | tail -1
```

### Вручную:
1. Просмотреть все файлы в `docs/3. tasks/` и подпапках `Done/`
2. Найти максимальный номер `TNN`
3. Использовать следующий номер (максимальный + 1) для новой задачи

---

## 4. Размещение задач

### Текущие задачи (в работе)
```
docs/3. tasks/SA-TNN_sNN_<snake_case>.md
```

### Завершённые задачи
```
docs/3. tasks/Done/SA-SNN_<имя>_done/SA-TNN_sNN_<snake_case>_done.md
```

**Пример структуры:**
```
docs/3. tasks/Done/
├── SA-S01_fts_search_done/
│   ├── SA-T001_s01_fts_indexing_done.md
│   └── SA-T002_s01_fts_search_api_done.md
└── SA-S02_web_dashboard_done/
    └── SA-T003_s02_datasette_config_done.md
```

### Отменённые задачи
```
docs/3. tasks/Done/SA-SNN_<имя>_done/SA-TNN_sNN_<snake_case>_cancelled.md
```

## 5. Ключевые документы в корне docs/

| Файл | Назначение |
|------|-----------|
| `SA-architecture.md` | Техническая архитектура, DB схема, data flow |
| `SA-HANDOFF.md` | Текущее состояние проекта, следующие шаги |
| `SA-CHANGELOG.md` | История изменений (Keep a Changelog) |

---

## 6. Процедура переноса задачи (обязательно)

1. **Перенести файл** задачи в целевую папку Done
2. **Добавить первой строкой** ссылку на спецификацию
3. **Обновить ссылки** в HANDOFF.md и CHANGELOG.md
4. **Удалить исходный файл** из `docs/3. tasks/`

### Мини-чеклист
- [ ] Старый файл удалён
- [ ] Ссылки обновлены (SA-CHANGELOG/SA-HANDOFF)
- [ ] Grep по старому пути — пусто

---

## 7. Правила для AI-ассистентов

### При создании новой задачи:
1. Всегда выполнять поиск максимального номера `TNN` среди всех файлов в `docs/3. tasks/` (включая `Done/`)
2. Не полагаться на `SA-CHANGELOG.md` для определения следующего номера
3. Проверять на конфликты

### При завершении задачи:
1. Перенести файл в `docs/3. tasks/Done/SA-SNN_*_done/` с суффиксом `_done.md`
2. Добавить ссылку на спецификацию первой строкой
3. Обновить SA-CHANGELOG.md и SA-HANDOFF.md

### Нумерация задач
Задачи имеют **только числовые ID** (SA-T001, SA-T002...).
**ЗАПРЕЩЕНО** добавлять буквы/суффиксы к номерам (SA-T001a, SA-T001b — ЗАПРЕЩЕНО).
При разбиении задачи на подзадачи — перенумеровывать со следующего свободного номера.

---

## 8. Общие правила

- **Всегда используйте текущую максимальную `NN`** и увеличивайте на 1 для новых файлов
- **Нумерация сквозная:** номера задач не привязаны к номерам спецификаций
- **Не дублируйте историю:** SA-HANDOFF.md — текущее состояние, SA-CHANGELOG.md — полная история

## Примеры

### Правильное именование

✅ `SA-S01_fts_search.md`
✅ `SA-T001_s01_fts_indexing.md`
✅ `SA-T002_s01_fts_search_api_done.md`

### Неправильное именование

❌ `S01_fts.md` (нет префикса SA-)
❌ `SA-T001a_fts.md` (буква в номере)
❌ `SA-T1_fts.md` (однозначный номер вместо трёхзначного)
❌ `SA-T01_ФТС-поиск.md` (кириллица и дефисы)
